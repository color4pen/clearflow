```yaml
概要:
  title: エラーポリシー
  purpose: プロジェクト横断のエラー分類・HTTPステータスコードマッピング・リトライ・ロールバック方針を定義する

ステータスコードマッピング:
  - cond: バリデーションエラー（入力値不正）
    status: 400
    exception: ValidationError
  - cond: 認証トークンなし・無効
    status: 401
    exception: AuthenticationError
  - cond: 権限不足
    status: 403
    exception: PermissionError
  - cond: 対象リソースが存在しない
    status: 404
    exception: NotFoundError
  - cond: 楽観的排他ロック競合
    status: 409
    exception: ConflictError
  - cond: 在庫不足・ビジネスルール違反
    status: 422
    exception: DomainError
  - cond: 外部API・DBタイムアウト
    status: 503
    exception: ExternalServiceError
  - cond: 上記以外の予期せぬエラー
    status: 500
    exception: InternalError

レスポンス形式:
  body:
    error:
      code: "{例外型をスネークケースで。例: not_found_error}"
      message: "{人間が読めるメッセージ。スタックトレース禁止}"
      detail: "{バリデーションエラー時のみ。フィールドごとのエラー配列}"

リトライポリシー:
  対象: ExternalServiceError のみ（冪等な操作に限る）
  上限回数: 3
  間隔: 指数バックオフ（1s / 2s / 4s）
  非対象: DomainError / ValidationError / ConflictError（リトライしても解消しない）

ロールバック方針:
  - cond: tx 内で例外発生
    action: DB トランザクション全体をロールバックする
  - cond: 外部API呼び出し後に例外発生
    action: 補償トランザクション（補償処理は各 UC のフローに明記する）
  - cond: 非同期ジョブ失敗
    action: デッドレターキューへ移動・アラート通知

ログポリシー:
  - level: ERROR
    when: 500 / 503 系
    include: [request_id, user_id, stack_trace]
  - level: WARN
    when: 409 / 422 系
    include: [request_id, user_id, error_code]
  - level: INFO
    when: 400 / 401 / 403 / 404 系
    include: [request_id, error_code]
  禁止: レスポンスボディへのスタックトレース出力

補足:
  - 各 UC の 公開IF.エラー はこのポリシーのステータスコードに従う
  - 新しい例外型を追加する場合はこのポリシーを先に更新する
```
