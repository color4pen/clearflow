# 契約・請求画面のデザイン適用

## Meta

- **type**: spec-change
- **slug**: design-contract-invoice
- **base-branch**: main
- **adr**: false

## 背景

Claude Design（docs/design/Clearflow.dc.html の CONTRACTS LIST / CONTRACT DETAIL / INVOICE DETAIL セクション）に合わせて契約・請求画面のレイアウトを更新する。

## 現状コードの前提

- `src/app/(dashboard)/contracts/page.tsx` — 契約一覧。5 カラム（契約名, 顧客名, 契約形態, 金額, ステータス）。案件名・期間カラムなし
- `src/app/(dashboard)/contracts/[id]/page.tsx` — 契約詳細。現在のレイアウト構成を実装時に確認する
- `src/app/(dashboard)/contracts/[id]/invoices/[invoiceId]/page.tsx` — 請求詳細
- `docs/design/Clearflow.dc.html` — 契約一覧: 7 カラム（契約名, 顧客名, 案件名, 契約形態, 金額, 期間, ステータス）。契約詳細: 2 カラム（左: 基本情報 + ステータス操作、右: 請求リスト + サマリ）。請求詳細: max-width 560px の狭幅レイアウト

## 要件

1. **契約一覧 — カラム追加**: 案件名と期間カラムを追加して 7 カラムにする。終了日が 30 日以内の行をハイライト
2. **契約詳細 — 2 カラムレイアウト**: 左カラム（1.5fr）に基本情報（読み取り表示 + 編集ボタン）+ ステータス操作 + 案件・顧客リンク。右カラム（1fr）に請求リスト + 請求サマリ（単発契約: プログレスバー風の契約金額/請求済/残り表示）+ 請求作成ボタン。承認待ちバナー（該当時）
3. **請求詳細 — 狭幅レイアウト**: max-width 560px のセンタリングされたレイアウト。90px ラベル + 1fr 値の grid。ステータス操作ボタン。入金確認ダイアログ（入金日入力）。パンくず（契約一覧 > 契約名 > 請求詳細）

## スコープ外

- 契約・請求の登録フォームのデザイン変更
- ビジネスロジックの変更

## 受け入れ基準

- [ ] 契約一覧が 7 カラムになっている
- [ ] 終了日 30 日以内の行がハイライトされている
- [ ] 契約詳細が 2 カラムレイアウトになっている
- [ ] 請求サマリにプログレスバー風の表示がある（単発契約）
- [ ] 請求詳細が max-width 560px の狭幅レイアウトになっている
- [ ] `typecheck && test` が green

## architect 評価済みの設計判断

1. **請求詳細を狭幅レイアウトに** — 請求は情報量が少ないため全幅は不要。560px に制限して読みやすくする
