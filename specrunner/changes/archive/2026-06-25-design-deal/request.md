## 背景

Claude Design（docs/design/Clearflow.dc.html の DEALS LIST / DEAL DETAIL セクション）に合わせて案件一覧・詳細画面のレイアウトを更新する。

## 現状コードの前提

- `src/app/(dashboard)/deals/page.tsx` — 一覧ページ。PageToolbar + フェーズタブ（Link ベース）+ DataTable。5カラム（フェーズ, 案件名, 顧客名, 担当者, 作成日）。パイプラインサマリなし。顧客・契約形態フィルタなし
- `src/app/(dashboard)/deals/[id]/page.tsx` — 詳細ページ。grid-cols-2（50/50）+ gap-2。左: DealInfoSection（フォーム形式、フェーズ変更はドロップダウン統合）+ 関連情報（dl 形式）。右: 契約セクション + 担当者 + 商談 + アクションアイテム + メモ
- `src/app/(dashboard)/deals/[id]/DealInfoSection.tsx` — フォーム形式の編集。フェーズ変更が select ドロップダウンで統合
- `src/app/(dashboard)/deals/[id]/DealContactsSection.tsx` — 担当者テーブル（名前, 部署, 役職, 役割の 4 カラム）+ 削除ボタン + 追加フォーム
- `src/app/(dashboard)/deals/[id]/DealActionItemsSection.tsx` — アクションアイテム（チェックボックス + テキスト + 担当者 + 期日）
- `docs/design/Clearflow.dc.html:367-420` — デザインの一覧: パイプラインサマリ（6カラム）+ 3 つの select フィルタ + 6 カラムテーブル（案件名, 顧客名, フェーズ, 契約形態, 想定金額, 受注見込み）
- `docs/design/Clearflow.dc.html:422-488` — デザインの詳細: パンくず + タイトル + 受注/失注ボタン + 1.5fr:1fr の 2 カラム。左: 基本情報（読み取り）+ 関連リンク + フェーズ変更ボタン群 + 商談記録。右: 契約（緑ヘッダー）+ 担当者（シンプルな名前+役割）+ アクションアイテム

## 要件

1. **一覧 — パイプラインサマリ**: 6 カラムグリッド（5 フェーズ + 合計）をテーブル上部に追加する。ダッシュボードと同じスタイル。セルクリックでフェーズフィルタを適用
2. **一覧 — フィルタ変更**: フェーズタブを 3 つの select ドロップダウン（フェーズ / 顧客 / 契約形態）に変更する
3. **一覧 — テーブルカラム**: 5 カラム（案件名, 顧客名, フェーズ, 契約形態, 想定金額）に変更する。DataTable コンポーネントを引き続き使用する（grid-template-columns ではなく DataTable の columns 定義で制御）。担当者カラムを削除。想定金額は mono フォント右寄せ。「受注見込み」カラムはデータソース（expectedCloseDate）の存在を実装時に確認し、存在すれば追加する
4. **詳細 — レイアウト変更**: grid-cols-2 gap-2 を `grid-template-columns: 1.5fr 1fr; gap: 24px` に変更する
5. **詳細 — ヘッダー**: パンくず（案件一覧 / 案件名）+ タイトル + サブテキスト（顧客名 + フェーズ）。右側に「受注にする」（緑ボーダー）+ 「失注にする」（赤ボーダー）ボタン。won/lost の場合はボタン非表示
6. **詳細 — 基本情報**: フォーム形式から読み取り表示（90px ラベル + 1fr 値のグリッド）に変更する。編集は「編集」ボタンクリックで DealInfoSection のフォームモードに切り替える（useState で表示/編集を切替。クリックで編集のインライン方式ではなく、セクション全体をフォームに切り替える）
7. **詳細 — フェーズ変更**: select ドロップダウンから独立セクションのボタン群に変更する。各フェーズがボタンで、現在フェーズがハイライト。クリックでフェーズ変更
8. **詳細 — 担当者セクション簡素化**: 4 カラムテーブルから、名前 + 役割ラベルの flex レイアウトに変更する。追加フォームは維持
9. **詳細 — 契約セクション**: ヘッダーに緑背景（bg-success-light #eef7f1）、テキスト色 #1a8a4a を適用する
10. **詳細 — 商談記録**: 左カラムに配置する（現在は右カラム）。行のスタイルを種別タグ + 日時で統一する

## スコープ外

- 案件登録フォームのデザイン変更
- ビジネスロジック・ユースケースの変更
- レスポンシブ対応
- DataTable コンポーネント自体の grid-template-columns 化（既存の table 要素ベースを維持）

## 実装方針（request-review 指摘への回答）

- **受注見込みカラム**: Deal モデルに `expectedCloseDate` が存在すれば表示する。実装時に `src/domain/models/deal.ts` を確認し、存在しなければカラムを省略する
- **編集モード**: 「クリックで編集」のインライン方式ではなく、セクションヘッダーに「編集」ボタンを配置し、クリックで DealInfoSection 全体をフォームモードに切り替える。useState の isEditing フラグで表示/編集を切替する
- **DataTable との整合**: DataTable は既存の HTML table ベースを維持する。grid-template-columns は DataTable のリファクタリングが必要なため使用しない。カラム幅は DataTable の width プロパティで調整する
- **DealPhaseActions の責務**: 受注/失注ボタンをヘッダーに移動する。DealPhaseActions コンポーネントは非終端フェーズのボタン群を描画するコンポーネントとして残す。受注/失注の確認ダイアログは既存のものを再利用する

## architect 評価済みの設計判断

1. **フォーム→読み取り表示に変更** — デザインがラベル+値のグリッド表示を採用しているため合わせる。編集はクリックでインラインフォームに切り替えるか、モーダルで行う。却下案: フォーム維持 — デザインと乖離する
2. **フェーズ変更をボタン群に** — デザインが各フェーズをクリッカブルなボタンとして表示しているため合わせる。select よりも現在のフェーズが視覚的に把握しやすい。却下案: select 維持 — デザインと乖離する
