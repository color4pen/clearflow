# 承認画面のデザイン適用

## Meta

- **type**: spec-change
- **slug**: design-approval
- **base-branch**: main
- **adr**: false

## 背景

Claude Design（docs/design/Clearflow.dc.html の APPROVALS LIST / APPROVAL DETAIL セクション）に合わせて承認一覧・詳細画面のレイアウトを更新する。

## 現状コードの前提

- `src/app/(dashboard)/requests/page.tsx` — 承認一覧。BulkApprovalPanel を含むカスタムレイアウト
- `src/app/(dashboard)/requests/[id]/page.tsx` — 承認詳細
- `src/app/(dashboard)/requests/new/page.tsx` — 手動申請
- `docs/design/Clearflow.dc.html` — 承認一覧: タブ（要対応/自分の申請/すべて）+ 5 カラムテーブル（件名, 申請者, ステータス, 手動/自動, 申請日）。承認詳細: ヘッダー + システム連動バナー + フォームデータ + ステップ進捗（縦ステッパー）+ 承認/却下ボタン

## 要件

1. **一覧 — タブ切替**: 「要対応」「自分の申請」「すべて」のタブを追加する。デフォルトタブはロールに応じて切替（member: 自分の申請、それ以外: 要対応）。「すべて」は admin/manager のみ
2. **一覧 — テーブル**: 5 カラム（件名, 申請者, ステータスバッジ, 手動/自動ラベル, 申請日）。grid 比率: `1.9fr 90px 110px 90px 110px`
3. **詳細 — ヘッダー**: 件名 + ステータスバッジ + 申請者 + 日時
4. **詳細 — システム連動バナー**: origin_type = system の場合、「この承認は引合「○○」の案件化に必要です」+ リンク
5. **詳細 — ステップ進捗**: 縦のステッパー UI。各ステップ: ステップ名、承認者、状態アイコン（待機/承認済/却下）、コメント、日時。現在のステップをハイライト
6. **詳細 — 承認/却下操作**: 該当ステップの承認者の場合のみ表示。コメント入力 + 「承認する」（primary）+ 「却下する」（danger outline）ボタン

## スコープ外

- 手動申請フォームのデザイン変更
- BulkApprovalPanel の廃止（一覧タブに統合するが既存の一括承認機能は維持）
- ビジネスロジックの変更

## 受け入れ基準

- [ ] 一覧にタブ切替（要対応/自分の申請/すべて）がある
- [ ] テーブルが 5 カラムで手動/自動ラベルが表示される
- [ ] 詳細にステータスバッジ付きヘッダーがある
- [ ] system origin の場合にシステム連動バナーが表示される
- [ ] ステップ進捗が縦ステッパー UI で表示される
- [ ] 承認/却下ボタンが該当承認者にのみ表示される
- [ ] `typecheck && test` が green

## architect 評価済みの設計判断

1. **ステッパー UI の新設** — 既存のテーブル表示からステッパー（縦のタイムライン風）に変更する。承認フローの進行状況が直感的に把握できる
