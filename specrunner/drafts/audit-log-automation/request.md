## 背景

監査ログの記録が 37 箇所のユースケースに手動でコピペされている。記録漏れのリスクがあり、新しいユースケースを追加するたびに監査ログの呼び出しを忘れないようにする必要がある。

R01 で導入したドメインイベント基盤の同期ハンドラとして監査ログ記録を実装し、ユースケースからの手動呼び出しを不要にする。

本リクエストは R01（ドメインイベント基盤）の完了を前提とする。

## 現状コードの前提

- `src/application/usecases/` 配下の 37 箇所に `auditLogRepository.create(...)` の呼び出しがある
- 各呼び出しは action, targetType, targetId, actorId, organizationId, metadata を手動で指定している
- 監査ログはユースケースのトランザクション内で記録されている

## 要件

1. **監査ログハンドラの実装**: ドメインイベントの同期ハンドラとして監査ログ記録を実装する。各イベントに対応する action, targetType のマッピングを定義する
2. **イベント→監査ログのマッピング**: InquiryConverted → action: "converted", targetType: "inquiry" など、全ドメインイベントに対する監査ログのフィールドを定義する
3. **既存ユースケースの監査ログ呼び出し削除**: ドメインイベントを発行しているユースケースから手動の `auditLogRepository.create` 呼び出しを削除する
4. **イベント未発行のユースケースは手動呼び出しを維持**: R01 でイベント発行を組み込んでいないユースケース（CRUD の一部）は従来通り手動で監査ログを記録する。順次イベント化する際に削除する

## スコープ外

- 監査ログのスキーマ変更
- 監査ログ UI の変更
- イベント未発行ユースケースへのイベント追加

## 受け入れ基準

- [ ] ドメインイベント発行済みの全ユースケースで監査ログが自動記録される
- [ ] 該当ユースケースから手動の auditLogRepository.create 呼び出しが削除されている
- [ ] 監査ログの内容（action, targetType, metadata）が変更前と同一である
- [ ] イベント未発行のユースケースの監査ログが引き続き正常に記録される
- [ ] `typecheck && test` が green
