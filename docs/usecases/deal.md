# deal

### 案件を直接作成する

- **目的**: 既存顧客からの直接依頼を案件として管理する
- **前提**: なし（引き合いを経由しない）
- **権限**: admin / manager
- **結果**: 案件が作成される
- **制約**: 顧客の指定が必須。新規顧客をその場で作成できる
- **関連**: client

### 案件のフェーズを変更する

- **目的**: 営業活動の進捗に合わせてフェーズを更新する
- **前提**: 案件が受注・失注でないこと
- **権限**: admin / manager
- **結果**: フェーズが変更される
- **制約**: 受注・失注は不可逆。変更時に確認を求める
- **関連**: なし

### 案件情報を編集する

- **目的**: 案件の詳細情報（金額・期間・種別・担当者等）を更新する
- **前提**: 案件が存在すること
- **権限**: admin / manager
- **結果**: 案件情報が更新される
- **関連**: なし

### 案件の備考を更新する

- **目的**: 案件の状況や共有事項を記録・更新する
- **前提**: 案件が存在すること
- **権限**: admin / manager
- **結果**: 備考が更新される。詳細画面からインラインで編集できる
- **関連**: なし

### 案件を削除する

- **目的**: 間違えて作成した案件を消す
- **前提**: 案件に商談・契約が紐づいていないこと
- **権限**: admin / manager
- **結果**: 案件が削除される。引き合い経由の案件の場合、引き合いのステータスが対応中に戻る
- **制約**: 商談または契約が存在する案件は削除できない
- **関連**: inquiry, meeting, contract

### 案件の担当者を管理する

- **目的**: 案件に関わる顧客側の担当者と役割を記録する
- **前提**: 案件に顧客が紐づいていること
- **権限**: 全ロール
- **結果**: 担当者が追加・削除される
- **関連**: client
