# notification

### 通知を確認する

- **目的**: 自分がウォッチしている案件の更新に気づく
- **前提**: 案件をウォッチしていること
- **権限**: 全ロール（認証済みユーザー）。各ユーザーは自分宛の通知のみ閲覧できる
- **結果**: ウォッチ中の案件配下の更新（他ユーザーによる操作）が新しい順に表示され、未読件数が分かる
- **制約**: 通知対象の操作は限定される（案件の更新・フェーズ変更、商談・アクションアイテム・契約の追加）。自分の操作は含まれない
- **関連**: watch, deal

### 通知を既読にする

- **目的**: 確認済みの通知を未読から外す
- **前提**: なし
- **権限**: 全ロール（認証済みユーザー）
- **結果**: 自分の通知がすべて既読になり、未読件数が 0 になる
- **関連**: watch
