# contract

### 契約を作成する

- **目的**: 受注した案件に対して契約情報を登録する
- **前提**: 案件が受注フェーズであること
- **権限**: admin / manager
- **結果**: 契約が作成される。案件の情報（金額・種別・期間）が初期値として引き継がれる
- **制約**: 1案件に複数契約を作成できる（フェーズ分割契約等）
- **関連**: deal, client

### 契約情報を編集する

- **目的**: 契約の詳細情報（金額・期間・支払条件・更新タイプ等）を更新する
- **前提**: 契約が存在すること
- **権限**: admin / manager
- **結果**: 契約情報が更新される
- **関連**: なし

### 契約を完了する

- **目的**: 納品・検収が完了した契約、または期間満了の契約を終了する
- **前提**: 契約が有効であること
- **権限**: admin / manager
- **結果**: 契約が完了状態になる
- **関連**: なし

### 契約を解約する

- **目的**: 有効な契約を途中で打ち切る
- **前提**: 契約が有効であること
- **権限**: admin / manager
- **結果**: 契約が解約状態になる
- **関連**: なし

### 契約を削除する

- **目的**: 間違えて作成した契約を消す
- **前提**: 契約に請求が紐づいていないこと
- **権限**: admin / manager
- **結果**: 契約が削除される
- **制約**: 請求が存在する契約は削除できない
- **関連**: invoice

### 契約に関するやり取りを記録する

- **目的**: 契約内容の見直し・変更交渉などのやり取りを記録する
- **前提**: 契約が存在すること
- **権限**: admin / manager / member
- **結果**: kind = contract_adjustment の顧客接点が作成される
- **関連**: interaction
