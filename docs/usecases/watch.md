# watch

### 案件をウォッチする

- **目的**: 関心のある案件の更新を継続的に追えるよう購読する
- **前提**: 案件が自組織に存在すること
- **権限**: 全ロール（認証済みユーザー）
- **結果**: ログインユーザーがその案件をウォッチし、以後その案件配下の更新を通知で受け取れるようになる
- **制約**: 同一ユーザーが同一案件を重複してウォッチしても二重登録されない（冪等）
- **関連**: deal

### 案件のウォッチを解除する

- **目的**: 不要になった案件の通知を止める
- **前提**: なし
- **権限**: 全ロール（認証済みユーザー）
- **結果**: ログインユーザーのその案件へのウォッチが解除される
- **関連**: deal
