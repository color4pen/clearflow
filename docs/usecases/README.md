# ユースケース定義

## 運用ルール

- 1ドメイン1ファイル（`inquiry.md`, `deal.md` 等）
- フロントマターの `status` で状態管理: `draft`（検討中） / `confirmed`（確定済み）
- draft で検討・レビューし、合意が取れたら `status: confirmed` に変更する
- confirmed になったユースケースは spec-runner リクエストの根拠として使用する
- 実装が完了しても削除しない（ユースケースの正本として維持）

## 書き方

```yaml
---
status: draft
domain: inquiry
updated: 2026-06-21
---
```

### 構成

1. **ステータス/フェーズ定義** — 全ステータスの一覧と意味
2. **遷移ルール** — from/to/トリガー/権限/処理の表
3. **ユースケース詳細** — 各操作の前提条件・処理・事後条件
4. **未解決事項** — 検討中の判断（draft 時のみ）
