# inquiry

### 引き合いを受け付ける

- **目的**: 外部からの問い合わせや相談を記録する
- **前提**: なし
- **権限**: 全ロール
- **結果**: 引き合いが作成される
- **制約**: 顧客は未定でもよい。新規顧客をその場で作成できる
- **関連**: client

### 引き合いを案件化する

- **目的**: 見込みのある引き合いを案件として管理対象にする
- **前提**: 引き合いが未対応または見送り状態であること
- **権限**: admin / manager
- **結果**: 案件が作成され、引き合いのステータスが案件化済みになる
- **制約**: 同一引き合いから複数案件は作成できない。顧客が未登録の場合はエラー
- **関連**: deal, client

### 引き合いを見送る

- **目的**: 追わないと判断した引き合いをゴミ箱に入れる
- **前提**: 引き合いが案件化されていないこと
- **権限**: 全ロール
- **結果**: 引き合いが見送り状態になる
- **関連**: なし

### 見送った引き合いの対応を再開する

- **目的**: 一度見送った引き合いを再び対応対象にする
- **前提**: 引き合いが見送り状態であること
- **権限**: 全ロール
- **結果**: 引き合いが未対応に戻る
- **関連**: なし

### 引き合いを編集する

- **目的**: 引き合いの情報（件名・概要・流入経路・顧客）を修正する
- **前提**: 引き合いが存在すること
- **権限**: 全ロール
- **結果**: 引き合い情報が更新される
- **関連**: client

### 引き合いを削除する

- **目的**: 間違えて作成した引き合いを消す
- **前提**: 引き合いに案件が紐づいていないこと
- **権限**: admin / manager
- **結果**: 引き合いが削除される
- **制約**: 案件が存在する引き合いは削除できない
- **関連**: deal
