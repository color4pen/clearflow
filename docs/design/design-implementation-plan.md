# デザイン適用リクエスト計画

Claude Design で作成した画面デザイン（Clearflow.dc.html）を Tailwind テーマベースで実装に反映する。

## 方針

- デザイン HTML のインラインスタイルを Tailwind ユーティリティクラスに変換する
- 新しい CSS 変数やカラートークンが必要な場合は globals.css の `@theme inline` に追加する
- 既存コンポーネント（SectionCard, PageToolbar, FormField 等）のスタイルをデザインに合わせて更新する
- 新しい共通コンポーネントが必要な場合は `src/app/components/` に追加する
- ページコンポーネントのレイアウト・構成をデザインに合わせて調整する

## デザインから抽出したトークン差分

### フォント
- デザイン: Noto Sans JP（現在: Arial → Noto Sans JP に変更）
- サイズ: 11px / 12px / 12.5px / 13px / 15px / 18px（現在は text-xs = 12px 中心）

### 角丸
- デザイン: border-radius 3px〜4px が主流（現在は rounded-none が多い → rounded-sm に統一）

### 間隔
- デザイン: padding 8〜14px, gap 8〜10px が主流

## リクエスト一覧

### D01: Tailwind テーマ・グローバルスタイルの更新

**目的**: globals.css のフォント・角丸・間隔をデザインに合わせ、Tailwind テーマのベースを整える

**スコープ**:
- フォントファミリーを Noto Sans JP に変更（Google Fonts の読み込み追加）
- デフォルトの角丸を rounded（4px）に設定
- フォームコンポーネント（FormField, MoneyInput）の rounded-none → rounded-sm に変更
- ベースのフォントサイズ・行間の調整
- 不足しているカラートークンの追加（あれば）

### D02: サイドバー・レイアウトの刷新

**目的**: ダッシュボードの layout.tsx をデザインに合わせる

**スコープ**:
- サイドバーの幅、パディング、フォントサイズ、アクティブ状態のスタイル
- ナビゲーションアイテムの hover / active スタイル
- ユーザー情報（名前、ロール）の表示
- 承認バッジの表示
- メインコンテンツエリアのスクロール制御

### D03: 共通コンポーネントの更新

**目的**: 既存の共通コンポーネントをデザインに合わせて更新

**スコープ**:
- PageToolbar: フォントサイズ、パディング、ボタン配置
- SectionCard: ボーダー、角丸、パディング
- DataTable: ヘッダー背景色、行ホバー、セル間隔、フォントサイズ
- FormField / Input / Select / Textarea: ボーダー、角丸、フォントサイズ、フォーカス状態
- LinkButton: スタイルバリエーション（primary / outline / ghost）
- SubmitButton: サイズ、パディング、角丸
- ステータスバッジ: デザインのラベルスタイルに合わせる

### D04: ダッシュボード画面の適用

**目的**: 営業ダッシュボードと経理ダッシュボードをデザインに合わせる

**スコープ**:
- パイプラインサマリのカードレイアウト
- アクション待ちリストの行レイアウト（ラベル + 件名 + 日付）
- 停滞案件セクション
- 直近の活動セクション
- 経理ダッシュボードの請求一覧レイアウト

### D05: 引合画面の適用

**目的**: 引合一覧・詳細・登録フォームをデザインに合わせる

**スコープ**:
- 一覧: フィルタバー、テーブルスタイル
- 詳細: 2カラムレイアウト、ステータス操作ボタン、承認待ちバナー
- 登録フォーム: フィールド配置、顧客インライン登録

### D06: 案件画面の適用

**目的**: 案件一覧・詳細・登録フォームをデザインに合わせる

**スコープ**:
- 一覧: フェーズサマリ、テーブルスタイル
- 詳細: 2カラム、フェーズ変更コントロール、担当者セクション、アクションアイテム集約、契約セクション

### D07: 商談・顧客画面の適用

**目的**: 商談詳細・顧客一覧/詳細をデザインに合わせる

**スコープ**:
- 商談詳細: Markdown エディタ/プレビュー、出席者リスト、アクションアイテム
- 顧客一覧: テーブルスタイル
- 顧客詳細: 2カラム、担当者リスト、関連エンティティリスト

### D08: 契約・請求画面の適用

**目的**: 契約一覧/詳細、請求詳細/登録をデザインに合わせる

**スコープ**:
- 契約一覧: 期限切れハイライト
- 契約詳細: 請求サマリのプログレスバー
- 請求詳細: ステータス操作ボタン、入金確認ダイアログ
- 請求登録: 残り請求可能金額の表示

### D09: 承認画面の適用

**目的**: 承認一覧・詳細・手動申請をデザインに合わせる

**スコープ**:
- 一覧: タブ切替、ステータスバッジ
- 詳細: ステップ進捗のステッパーUI、承認/却下ボタン、コメント入力
- 手動申請: テンプレート選択、動的フォーム

### D10: 売上画面の適用

**目的**: 売上ダッシュボード・明細・予実管理をデザインに合わせる

**スコープ**:
- 売上ダッシュボード: 月次推移グラフ、顧客別ランキング
- 明細: 集計軸切替、テーブル
- 予実管理: プログレスバー、目標設定

### D11: 設定画面の適用

**目的**: 設定画面（ポリシー、テンプレート、委任、ユーザー、Webhook、監査ログ）をデザインに合わせる

**スコープ**:
- SettingsNav のタブスタイル
- 各設定画面のフォーム・テーブルレイアウト
- トグルスイッチ（ポリシー有効/無効）
- テンプレートのステップ定義 UI

### D12: ダイアログ・トーストの統一

**目的**: 確認ダイアログとトースト通知のスタイルを統一する

**スコープ**:
- 確認ダイアログ（受注/失注、契約完了/解除、削除）
- トースト通知（成功/エラー）
- 入金日入力ダイアログ

## 依存関係

```
D01 テーマ基盤
 └→ D02 レイアウト
     └→ D03 共通コンポーネント
         ├→ D04 ダッシュボード
         ├→ D05 引合
         ├→ D06 案件
         ├→ D07 商談・顧客
         ├→ D08 契約・請求
         ├→ D09 承認
         ├→ D10 売上
         ├→ D11 設定
         └→ D12 ダイアログ・トースト
```

D01 → D02 → D03 は直列。D04〜D12 は D03 完了後に並行実行可能。
