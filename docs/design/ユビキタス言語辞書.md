# ユビキタス言語辞書

ドメイン用語の正本。日本語表記・コード識別子・意味を 1 対 1 で定める。UI・設計書・会話はこの日本語表記に統一し、コードはこのコード識別子に統一する。

## エンティティ

| 用語（日本語） | コード識別子（テーブル / 型） | 意味 |
|---|---|---|
| 引合 | inquiries / Inquiry | 顧客からの問い合わせ。案件化前のパイプラインの入口 |
| 案件 | deals / Deal | 受託案件。引合から案件化、または直接作成する。パイプラインの中核 |
| 商談 | meetings / Meeting | 案件または引合の下で行う打ち合わせの記録。種別はヒアリング / 提案 / 交渉 / クロージング / フォローアップ |
| 契約 | contracts / Contract | 案件の受注後に結ぶ契約 |
| 請求 | invoices / Invoice | 契約に対する請求 |
| 顧客 | clients / Client | 取引先企業 |
| 顧客担当者 | client_contacts / ClientContact | 顧客側の担当者（個人） |
| 案件担当者 | deal_contacts / DealContact | 案件に紐づく顧客側の担当者（役割付き） |
| アクションアイテム（タスク） | action_items / ActionItem | 案件・商談・引合に紐づくタスク。UI 上の呼称は「タスク」 |
| 承認リクエスト | requests / Request | 承認ワークフローの申請 |
| 承認ステップ | approval_steps / ApprovalStep | 承認リクエスト内の各承認段階 |
| 承認テンプレート | approval_templates / ApprovalTemplate | 承認フローのひな形 |
| 承認ポリシー | approval_policies / ApprovalPolicy | ドメインアクションに応じて承認を自動起動する条件定義 |
| 承認委任 | approval_delegations / ApprovalDelegation | 承認権限の一時委任 |
| 売上目標 | revenue_targets / RevenueTarget | 期間ごとの売上目標 |
| ウォッチ | watches / Watch | ユーザーが案件を購読し、配下の更新を通知で受け取る登録（UI ボタン表記は Watch / Watching） |
| 監査ログ | audit_logs / AuditLog | 全状態変更の追記専用記録 |

## 読み取りモデル（派生情報）

監査ログなどから読み取り時に構成する情報。永続化された独自テーブルを持たない。

| 用語（日本語） | コード識別子（型） | 意味 |
|---|---|---|
| 案件アクティビティ | DealActivityResult | 案件配下の操作を監査ログから時系列に構成した活動履歴 |
| 通知 | DerivedNotification | ウォッチ中の案件配下の更新を監査ログから導出した通知 |

## 担当者の区別

| 区分 | 用語 | コード |
|---|---|---|
| 自社側 | 営業担当 | deal.assigneeId（ユーザー） |
| 自社側 | 技術担当 | deal.technicalLeadId（ユーザー） |
| 先方側 | 顧客担当者 / 案件担当者 | client_contacts / deal_contacts |

「担当者」とだけ書かず、自社側か先方側かを明示する。

## 表記の注意（混同しやすい用語）

- **商談 ≠ 会議**: `meeting` エンティティの日本語表記は必ず「商談」。種別が営業フェーズ（ヒアリング/提案/交渉/クロージング/フォローアップ）であり、社内会議ではない。日本語で「会議」と書かない（コードの英語識別子は歴史的経緯で `meeting` のまま）。
- **案件 ≠ 商談**: 「案件（deal）」はパイプラインの単位、「商談（meeting）」はその下の打ち合わせ記録。別物。
- **タスク = アクションアイテム**: UI 表記は「タスク」、ドメイン/コードは `action_item`。同一物。
