# 契約

## 契約一覧

### 要素

| 要素 | 内容 |
|---|---|
| 一覧テーブル（フィルタなし） | 全件表示 |
| 一覧テーブル | 契約名、顧客名、案件名、契約形態、金額、期間、ステータス |

### 状態

| 条件 | 表示 |
|---|---|
| end_date が 30 日以内 | 行をハイライト表示 |

### 遷移

| 操作 | 遷移先 |
|---|---|
| 行をクリック | 契約詳細 |

---

## 契約詳細

### 要素

| 要素 | 内容 |
|---|---|
| 基本情報 | 契約名、契約形態、金額、開始日、終了日、更新種別、ステータス。編集可能 |
| ステータス操作 | 現在のステータスに応じたアクションボタン |
| 案件リンク | 紐づく案件名と詳細へのリンク |
| 顧客リンク | 顧客名と詳細へのリンク |
| 請求リスト | 紐づく請求の一覧（金額、請求日、支払期日、ステータス） |
| 請求サマリ | 請求合計金額 / 契約金額（単発契約の場合） |
| 請求作成ボタン | |

### 状態

| 条件 | 表示 |
|---|---|
| status = active | 「完了する」「解除する」ボタンを表示 |
| status = completed / cancelled | ステータス操作ボタンなし |
| renewal_type = one_time | 請求サマリに「残り請求可能金額: ○○円」を表示 |
| 承認ポリシーにより承認待ち | 「承認待ちです」バナー |

### 操作

| 操作 | 権限 | 結果 |
|---|---|---|
| 基本情報を編集 | admin / manager / finance | 契約が更新される |
| 完了する | admin / manager / finance | status が completed になる（確認ダイアログ） |
| 解除する | admin / manager / finance | status が cancelled になる（確認ダイアログ）。承認ポリシー評価あり |
| 削除 | admin | 契約が削除される（確認ダイアログ） |

### 遷移

| 操作 | 遷移先 |
|---|---|
| 案件名クリック | 案件詳細 |
| 請求をクリック | 請求詳細 |
| 請求作成ボタン | 請求登録（contractId 設定済み） |

## 契約調整（やり取り）

契約詳細内のセクション。この契約に紐づく契約内容の見直し・変更交渉などのやり取りを記録・表示する。やり取りの記録は契約エンティティへの操作（編集・ステータス変更）とは別概念で、営業活動の延長として admin / manager / member が記録できる。

### 要素

| 要素 | 内容 |
|---|---|
| 調整タイムライン | この契約に紐づく `kind = contract_adjustment` の顧客接点を新しい順に表示（日時、要約、メモ） |
| 契約調整を記録ボタン | 日時・要約・任意のメモを入力して契約調整を記録する導線 |

### 操作

| 操作 | 権限 | 結果 |
|---|---|---|
| 契約調整を記録 | admin / manager / member | `kind = contract_adjustment`・contractId 設定済みの顧客接点が作成される |

## 契約登録

### 要素

| 要素 | 内容 |
|---|---|
| 契約名 | テキスト入力。必須 |
| 契約形態 | セレクト（contract_type）。必須。案件の想定契約形態をデフォルト値 |
| 金額 | 数値入力。必須 |
| 開始日 | 日付入力。必須 |
| 終了日 | 日付入力。任意 |
| 支払条件 | テキスト入力。任意 |
| 更新種別 | セレクト（one_time / recurring）。必須 |
| 更新サイクル | テキスト入力。任意。更新種別が recurring の場合のみ表示 |

**注**: dealId は遷移元から自動設定される。

### 遷移

| 操作 | 遷移先 |
|---|---|
| 登録成功 | 契約詳細（作成された契約） |
| キャンセル | 案件詳細 |
