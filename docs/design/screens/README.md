# 画面仕様

各画面の要素・状態・遷移を定義する。レイアウトや見た目は扱わない。

画面ごとに以下を記述する:

- **要素**: 画面に存在する情報とコントロール
- **状態**: データやユーザーの条件によって変化する表示
- **遷移**: この画面からどこに行けるか、どこから来るか
- **操作**: ユーザーが行える操作とその結果

## 共通要素

### グローバルナビゲーション

全画面で常に表示される。

| 要素 | 説明 |
|---|---|
| ナビゲーションメニュー | ダッシュボード / 顧客 / 引き合い / 案件 / タスク / 契約 / 売上 / 申請一覧 / 設定 / 監査ログ（admin のみ） |
| ログインユーザー情報 | ユーザー名、ロール |
| 通知 | サイドバー下部のベル。未読がある場合は件数をバッジ表示。クリックで通知パネルを開き、ウォッチ中の案件配下の更新（他ユーザーの操作）を新しい順に表示。「既読にする」で全件既読化 |

### パンくずリスト

一覧→詳細→子エンティティ のような階層遷移がある画面で表示する。

### トースト通知

作成・更新・削除・ステータス変更の操作結果をフィードバックする。成功時と失敗時で区別する。
