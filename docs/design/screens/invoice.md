# 請求

## 請求詳細

### 要素

| 要素 | 内容 |
|---|---|
| 基本情報 | タイトル、金額、請求日、支払期日、入金日、ステータス、備考。読み取り表示 |
| ステータス操作 | 現在のステータスに応じたアクションボタン |
| 契約リンク | 紐づく契約名と詳細へのリンク |

### 状態

| 条件 | 表示 |
|---|---|
| status = scheduled | 「発行する」ボタン |
| status = invoiced | 「入金確認」「期日超過にする」ボタン |
| status = overdue | 「入金確認」ボタン。超過日数を表示 |
| status = paid | 操作ボタンなし。入金日を表示 |
| 入金確認を押下 | 入金日の入力を求めるダイアログ |

### 操作

| 操作 | 権限 | 結果 |
|---|---|---|
| 基本情報を編集 | admin / finance | 請求が更新される |
| 発行する | admin / finance | status が invoiced になる |
| 入金確認 | admin / finance | status が paid になる。paidAt が設定される |
| 期日超過にする | admin / finance | status が overdue になる |

---

## 請求調整（やり取り）

請求詳細内のセクション。この請求に紐づく請求内容の修正・調整交渉などのやり取りを記録・表示する。やり取りの記録は請求エンティティへの操作（編集・ステータス変更）とは別概念で、請求は経理領域であるため admin / manager / finance が記録できる。

### 要素

| 要素 | 内容 |
|---|---|
| 調整タイムライン | この請求（invoiceId）に紐づく顧客接点を新しい順に表示（日時、要約、メモ） |
| 請求調整を記録ボタン | 日時・要約・任意のメモを入力して請求調整を記録する導線 |

### 操作

| 操作 | 権限 | 結果 |
|---|---|---|
| 請求調整を記録 | admin / manager / finance | kind = note（既定）・invoiceId 設定済みの顧客接点が作成される |

## 請求登録

### 要素

| 要素 | 内容 |
|---|---|
| 金額 | 数値入力。必須 |
| 請求日 | 日付入力。任意 |
| 支払期日 | 日付入力。必須 |
| 残り請求可能金額 | 単発契約の場合に表示（契約金額 − 既存請求合計） |

**注**: contractId は遷移元から自動設定される。

### バリデーション

| ルール | エラーメッセージ |
|---|---|
| 請求日 > 支払期日 | 「支払期日は請求日以降にしてください」 |
| 単発契約で合計が契約金額を超過 | 「請求金額の合計が契約金額を超えます（残り: ○○円）」 |

### 遷移

| 操作 | 遷移先 |
|---|---|
| 登録成功 | 請求詳細（作成された請求） |
| キャンセル | 契約詳細 |
