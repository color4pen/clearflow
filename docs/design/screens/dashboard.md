# ダッシュボード

## 営業ダッシュボード

**表示条件**: ロールが member / manager / admin の場合のデフォルト画面。

### 要素

| 要素 | 内容 |
|---|---|
| アクション待ちリスト | 自分が対応すべき項目の一覧。以下を統合して期日の近い順に表示する |
| — 承認待ち | 自分が承認者の承認リクエスト（件名、申請者、申請日） |
| — アクションアイテム | action_items テーブルから自分が担当の未完了タスクを取得（内容、期日、紐づけ先） |
| — 未対応引合 | ステータスが new の引合（件名、経路、登録日） |
| パイプラインサマリ | フェーズごとの案件数と想定金額合計 |
| 直近の活動 | 組織内の最新操作ログ（操作者、操作内容、対象、日時） |

### 状態

| 条件 | 表示 |
|---|---|
| アクション待ちが 0 件 | 「対応が必要な項目はありません」 |
| ロールが manager / admin | パイプラインサマリに担当者別の内訳を追加表示。停滞案件リスト（N 日以上フェーズ変更なし）を表示 |

### 遷移

| 操作 | 遷移先 |
|---|---|
| アクション待ちの項目をクリック | 承認リクエスト詳細 / 案件詳細（アクションアイテム元）/ 引合詳細 |
| パイプラインのフェーズをクリック | 案件一覧（該当フェーズでフィルタ済み） |
| 活動ログの項目をクリック | 対象エンティティの詳細画面 |

## 経理ダッシュボード

**表示条件**: ロールが finance の場合のデフォルト画面。

### 要素

| 要素 | 内容 |
|---|---|
| 期日超過の請求 | status が overdue の請求一覧（タイトル、契約名、金額、支払期日、超過日数） |
| 未入金の請求 | status が invoiced の請求一覧（タイトル、契約名、金額、支払期日）。期日の近い順 |
| 今月の売上サマリ | 今月の入金確認済み合計金額 |
| 請求予定 | status が scheduled で issue_date が今月〜来月の請求一覧 |

### 遷移

| 操作 | 遷移先 |
|---|---|
| 請求をクリック | 請求詳細 |
| 売上サマリをクリック | 売上ダッシュボード |
