# 承認

## 承認リクエスト一覧

### 要素

| 要素 | 内容 |
|---|---|
| タブ | 「要対応」（自分が承認者）/ 「自分の申請」/ 「すべて」 |
| フィルタ | タブによるステータス切替（要対応 / 自分の申請 / すべて） |
| 一覧テーブル | 件名、申請者、ステータス、起動パターン（手動/自動）、申請日 |
| 一括承認パネル | 要対応タブでチェックボックスによる複数選択 + 一括承認ボタン（admin / manager のみ） |

### 状態

| 条件 | 表示 |
|---|---|
| デフォルトタブ | ロールが member: 「自分の申請」。それ以外: 「要対応」 |
| 「すべて」タブ | admin / manager のみ表示 |
| 起動パターンが system | アイコンまたはラベルで区別 |

### 遷移

| 操作 | 遷移先 |
|---|---|
| 行をクリック | 承認リクエスト詳細 |
| 手動申請ボタン | 手動申請画面 |

## 承認リクエスト詳細

### 要素

| 要素 | 内容 |
|---|---|
| ヘッダ | 件名、申請者、申請日時、ステータス |
| 起動元情報 | system の場合: トリガーアクション名、対象エンティティへのリンク |
| フォームデータ | 申請時の入力内容（キーと値の一覧） |
| ステップ進捗 | 全承認ステップの一覧（順序、ステップ名、承認者、ステータス、コメント、対応日時） |
| 承認/却下操作 | 現在のステップの承認者の場合に表示 |
| コメント入力 | 承認/却下時のコメント。任意 |

### 状態

| 条件 | 表示 |
|---|---|
| 自分が現在のステップの承認者（または委任先） | 「承認する」「却下する」ボタンを表示 |
| 自分が承認者でない | 操作ボタンなし。閲覧のみ |
| status = approved | 全ステップに承認済みマーク |
| status = rejected | 却下されたステップをハイライト |
| status = revision | 差し戻し状態。「再申請する」ボタンを表示 |
| status = expired | 期限切れの表示 |
| origin_type = system | 対象エンティティ（引合、契約等）へのリンクを表示 |

### 操作

| 操作 | 権限 | 結果 |
|---|---|---|
| 承認する | 該当ステップの承認者 / 委任先 | ステップが approved に。全ステップ完了ならリクエストが approved に。システム連動なら承認後アクション実行 |
| 却下する | 該当ステップの承認者 / 委任先 | リクエストが rejected になる |
| 再申請する | 申請者 | revision 状態のリクエストを pending に戻す |
| 一括承認 | admin / manager | 選択した複数リクエストを一括で承認する |

## 手動申請

### 要素

| 要素 | 内容 |
|---|---|
| テンプレート選択 | 利用可能なテンプレートのセレクト |
| タイトル | テキスト入力。必須 |
| フォーム | テンプレートの formFields に基づいて動的に生成されるフォーム |
| 申請ボタン | |

### 遷移

| 操作 | 遷移先 |
|---|---|
| 申請を作成 | 承認リクエスト詳細（status = pending） |
| キャンセル | 承認リクエスト一覧 |
